        → Validates webhook signature

        → Updates Payment status to Success
        → Updates BillingInvoice status to Paid
        → Updates Subscription dates if renewal
        → Creates audit log entry

        decimal amount)
        → Creates Payment record with Pending status
        → Returns paymentId

7.1  Registration Flow — Subscription Creation
     When a new company registers (AuthController.Register):
     a) Create Company record (already done)
     b) Create Subscription record:
        - PlanId = selected plan (or default Starter)
        - Status = Trial
        - StartDate = now
        - EndDate = now + 14 days (trial period)
     c) No invoice created during trial

7.2  Subscription Lifecycle — Background Job (or Login Check)
     On user login (AuthController.Login):
     a) Check company's active subscription
     b) If EndDate < now AND Status != Expired:
        - If within GracePeriodDays: set Status = GracePeriod
        - If past grace period: set Status = Expired
     c) Add subscription status to JWT claims
     d) This allows the Dashboard banner to show correctly

7.3  Invoice Generation
     a) When subscription is near expiry (via login check or scheduled):
        - Auto-create BillingInvoice for next billing period
        - Amount = Plan.Price
        - DueDate = Subscription.EndDate
        - Status = Unpaid
     b) Admin can also manually trigger renewal from Billing page

7.4  Payment Flow
     a) Admin clicks "Renew" or "Pay Invoice" on Billing page
     b) Web creates checkout via POST /api/billing/checkout
     c) API creates Payment record (Pending), calls PayMongo
     d) User redirected to PayMongo hosted checkout page
     e) On success, PayMongo calls webhook
     f) Webhook handler:
        - Marks Payment as Success
        - Marks Invoice as Paid
        - Extends Subscription.EndDate by billing cycle duration
        - Sets Subscription.Status = Active
        - Creates AuditLog entry

7.5  Plan Upgrade Flow
     a) Admin selects new plan on Billing page
     b) System creates prorated invoice (or full price for simplicity)
     c) Same checkout flow as renewal
     d) On success, update Company.PlanId and Subscription.PlanId
